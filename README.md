# Telegram Clicker Web App

Новый проект с нуля: `Telegram Web App` кликер с Node.js-ботом и встроенным HTTP-сервером для деплоя на Bothost.

## Что внутри

- большой тап-экран с мобильной вёрсткой под Telegram;
- магазин улучшений для усиления клика;
- магазин авто-кликеров для пассивного дохода;
- сохранение прогресса в `localStorage`;
- безопасная интеграция с Telegram `WebApp` API: `ready`, `expand`, haptic feedback и чтение пользователя;
- Node.js-бот, который отвечает на `/start` и открывает мини-приложение кнопкой `Web App`.

```powershell
cd C:\Users\natal\OneDrive\Документы\WEBapp\frontend
```

## Переменные окружения

- `BOT_TOKEN` - токен от `@BotFather`
- `WEBAPP_URL` - публичный `https://` адрес мини-приложения
- `PORT` - внутренний порт сервера, для Bothost оставь `3000`
- `BOT_LABEL` - подпись кнопки Web App в меню Telegram

## Bothost

Для Bothost используй такие значения:

- Платформа: `Telegram`
- Язык: `Node.js`
- Локация: `Нидерланды`
- Использовать домен: `Да`
- Порт веб-приложения: `3000`
- Главный файл: `server.js`
- Переменные окружения: минимум `BOT_TOKEN` и `WEBAPP_URL`

Если задаешь кастомный домен, то `WEBAPP_URL` должен быть полным адресом вида `https://treecubeclick.bothost.ru`.

## Примечания

- В обычном браузере мини-приложение тоже открывается и подходит для базового теста.
- При старте `server.js` сам проверяет сборку фронтенда и при необходимости собирает её.
- Прогресс пока хранится на устройстве в `localStorage`, без серверной синхронизации между телефонами.
